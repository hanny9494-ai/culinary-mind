# L0 Causal-Chain Schema — v1.1

Producer: `pipeline/l0/extract.py`
Sink: `output/{book_id}/stage4/l0_principles_open.jsonl`
      `output/{book_id}/stage4/stage4_dedup.jsonl`

Version field: `_v` (default `"1.0"` when absent).

## Fields

| Field                  | Type              | Required | Description                                                                                  |
|------------------------|-------------------|----------|----------------------------------------------------------------------------------------------|
| `_v`                   | string            | no       | Schema version. Default `"1.0"` when missing. Bumps only on breaking change.                 |
| `evidence_type` | enum string       | no       | Added in v1.1. One of `textbook`, `empirical`, `review`, `computed`. Marks how strongly to trust this record at inference time. |
| `scientific_statement` | string            | yes      | The canonical statement of the fact or causal chain.                                         |
| `proposition_type`     | enum string       | yes      | One of `fact_atom`, `causal_chain`, `compound_condition`, `mathematical_law`.                |
| `causal_chain_steps`   | array of string   | no       | For `causal_chain` types: ordered textual steps, e.g. `["Trigger", "Process", "Result"]`.    |
| `causal_chain_text`    | string            | no       | `A→B→C` one-line summary of the chain. Empty when `proposition_type != causal_chain`.       |
| `boundary_conditions`  | array of string   | no       | Plain-text constraints under which the statement holds.                                      |
| `boundary_zones`       | array of string   | no       | Structured zones when the boundary is multi-dimensional. Reserved — usually `[]` in v1.0.    |
| `domain`               | string            | yes      | One of the 17 L0 domains, or `other` / `unclassified`.                                       |
| `confidence`           | float (0.0–1.0)   | yes      | Model's confidence in the statement.                                                         |
| `citation_quote`       | string            | yes      | Verbatim or near-verbatim quote from the source page.                                        |
| `domain_note`          | string            | no       | Free-form clarification about the domain assignment.                                         |
| `source_chunk_id`      | string            | yes      | Book-local chunk identifier, e.g. `chunk_5`.                                                 |
| `dedup_status`         | enum string       | no       | Populated by the dedup pass: `novel`, `near_dup`, `exact_dup`.                               |
| `dedup_max_sim`        | float             | no       | Max cosine similarity against earlier records (dedup pass output).                           |

## Constraints

- `domain` MUST be drawn from the 17 L0 domains listed in `CLAUDE.md §2`;
  anything else falls under `other`.
- `confidence` is strictly `[0.0, 1.0]`.
- `causal_chain_text` and `causal_chain_steps` are co-required: if one
  is non-empty the other should be too.


## `evidence_type` (added in v1.1)

Optional enum describing the provenance quality of this record.

| Value      | Meaning                                                                       |
|------------|-------------------------------------------------------------------------------|
| `textbook` | From an authoritative textbook / monograph. Default for distilled book data.  |
| `empirical`| From an experimental study / peer-reviewed dataset. Primary evidence.         |
| `review`   | From a review paper aggregating other sources. Secondary evidence.            |
| `computed` | Derived by a solver / model / another record. Not primary source.             |

Backward compatibility: missing field → treat as `None` (unknown).
Consumers should not reject v1.0 records lacking this field.

  "causal_chain_text": "加热→蛋蛋白质凝固→液体增稠/凝固",
