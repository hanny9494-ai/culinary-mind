STEP 1 — Identity verification

Check zh ↔ scientific_name ↔ cuisine_deep ingredients alignment.
- If display_name_zh maps to a species different from scientific_name → emit `zh_sci_mismatch` and set tree_status=identity_conflict.
- If raw atom contains cuisine_deep references to a different taxon (e.g. pomfret with japanese pufferfish references) → emit `cuisine_deep_taxon_mismatch` (and `heavy_cross_contamination` if 2+ cross-taxon fields).
- For pomfret-style identity_conflict, prefer scientific_name as anchor and quarantine cuisine_deep contamination.

STEP 2 — Form type classification

- species: a single biological species used directly (chicken / beef / wheat).
- variety: a specific variety/breed/cultivar of a species (bresse_chicken / russet_potato / kobe_beef). Scientific_name MAY include the cultivar suffix (`Solanum tuberosum 'Russet Burbank'` or `Gallus gallus domesticus (Bresse Gauloise)`) — keep it.
- part: a specific anatomical part (chicken_skin / pork_belly / beef_chuck). PART_OF the parent species.
- processed: a transformation of a biological source (soy_sauce / oyster_sauce / dried_shrimp / chicken_stock / beef_jerky). See STEP 3.
- composite: multi-component prepared dish or composite product (chicken_curry / 17th_century-era food).
- ambiguous: ONLY when the atom_id genuinely refers to multiple distinct biological species in different culinary contexts (e.g. "black_bean" can mean Phaseolus vulgaris OR fermented Glycine max). Do NOT use ambiguous as a fallback for unfamiliar atoms — prefer needs_review.

STEP 3 — Processed-form rules (CRITICAL)

For sauce / stock / extract / oil / cream / paste / syrup / jam / powder / flour / jerky / vinegar / dried_*/cured_*/smoked_*/fermented_*/pickled_* atoms:
- form_type = processed (NEVER composite)
- scientific_name MUST preserve the biological source (e.g. `Bos taurus`, `Crassostrea spp.`, `Oryza sativa`). Do NOT use descriptive names like "Dehydrated beef" or "N/A".
- Process metadata goes in derived_from (process_type, source_canonical_id), NOT in scientific_name.
- For multi-source ferments (soy_sauce = Glycine max + Triticum aestivum), list both in scientific_name as `Genus species, Genus species` (no "(fermented)" suffix; the form_type already says processed).

STEP 3.5 — Compound dish suspect detection

If the canonical_id ends with `_soup_base`, `_curry`, `_stew`, `_aspic`, `_stock_base`, `_marinade`,
`_seasoning_blend`, or any compound multi-component preparation suffix, AND the raw atom mixes
multiple ingredient sources without a single dominant species:
- Set tree_status = needs_review
- Add issue_code: "suspect"
- Note that the atom is a derived compound product, not a single ingredient

Do NOT mark these excluded — they are legitimate culinary entities that simply require human
review before entering the active tree.

STEP 4 — Composition tolerance

Composition sum (water + protein + fat + carb + ash + fiber) is normally 95–105%. Three exceptions where lower sums are normal — DO NOT emit composition_implausible:
- Dehydration foods (dried_*, _powder, _jerky, _chips): 60–100% normal (residual salt/ash/fiber).
- Alcoholic beverages (_wine, _beer, _spirits, rice_wine): 80–95% normal (ethanol + dissolved solids).
- Fermented condiments (soy_sauce, fish_sauce, vinegar): 95–105% but high salt + water content.

STEP 5 — Cluster outlier check

If sibling_atoms_array is non-empty (atoms sharing the same scientific_name):
- If this atom's culinary_role / form_type contradicts the sibling consensus → emit `chimera_node` and set tree_status=needs_review.
- Otherwise note convergence in evidence_split_candidates if marginal.

STEP 6 — Output target_node

Emit the canonical schema. Keep aliases as a list of strings. Empty list is fine — don't pad.

For peak_season_codes, normalize the raw 303 messy values:
- "year-round" / "always" / "all year" → year_round
- "varies" / "varies regionally" → varies_by_region
- Single seasons → spring/summer/autumn/winter
- Months strings → also fill peak_months as integer array.

For dietary_flags, infer from scientific_name + form_type. Beef → halal/kosher. Pork → not halal/not kosher. Wheat-derived processed → not gluten_free.

For allergens, infer from species: tree_nut → nuts; shellfish → shellfish; milk/cream → dairy; egg → egg; wheat → gluten.

STEP 7 — confidence_overall

A float 0.0-1.0 reflecting overall confidence. ≥0.85 for clean atoms with clear identity; 0.50-0.80 for ambiguous; <0.50 for severe identity_conflict or heavy contamination.

STEP 8 — needs_human_review

True when:
- tree_status ∈ {identity_conflict, needs_review}
- confidence_overall < 0.70
- Heavy contamination (3+ cross-taxon fields) even if you reconciled them

False otherwise. Excluded nodes never need review (the pre-filter already decided).
