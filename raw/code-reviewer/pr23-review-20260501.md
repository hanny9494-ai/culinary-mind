## E. 必修清单（合入前必须完成）

### P1（blocker）

1. **G1/L1 — Tier 3 quarantine rename 失败时不要清空幂等状态**：
   ```ts
   // file-watcher.ts:246-272 — 修改建议
   if (count >= FileWatcher.MAX_RENAME_RETRIES) {
     const failedDir = join(getCeHubDir(), 'results', '_archive_failed');
     ensureDir(failedDir);
     const failedPath = uniqueArchivePath(failedDir, basename(filePath));
     try {
       this.safeRenameSync(filePath, failedPath);
       // 成功 quarantine：清状态 + return true
       appendRaw('archive_failures.jsonl', { source: filePath, target_dir: targetDir,
         quarantined_to: failedPath, error: errMsg, attempts: count, task_id: taskId, ts: Date.now() });
       this.failedRenameCount.delete(filePath);
       this.processedSideEffects.delete(filePath);
       return true;
     } catch (qErr) {
       // quarantine 也失败：保留幂等标记，记录但不重试 side effects
       const qMsg = qErr instanceof Error ? qErr.message : String(qErr);
       console.error(`[FileWatcher] quarantine rename also failed for ${filePath}: ${qMsg}`);
       appendRaw('archive_failures.jsonl', { source: filePath, target_dir: targetDir,
         quarantined_to: null, error: errMsg, quarantine_error: qMsg, attempts: count,
         task_id: taskId, ts: Date.now(), surrendered: true });
       // 关键：**不要** delete failedRenameCount / processedSideEffects
       // 让 sideEffectsAlreadyRun 永远 true，避免下轮 poll 重跑 side effects
       // 文件留在原位由人工/外部恢复
       return true;
     }
   }
   ```
   或者更彻底地把 truly-broken 文件名打上 `*.surrender.json` 的 sticky marker，让 `processExisting` 跳过 `*.surrender.json`。

2. **必须新增对应单元测试**：mock `safeRenameSync` 让 target 和 quarantine 路径都抛 EBUSY → 验证：

   - `appendRaw('archive_failures.jsonl')` 含 `surrendered: true` 或等价标记
   - 后续重复调用 handleResult 不再触发 side effects
