## A. P1 必修（GPT 5.5 主审独立发现，Opus 复核确认）

### P1.1 — M01 接受负浓度，违反物理约束

**文件:行号**：`engine/solver/mf_m01.py:51-52`

```python
val.require_finite("C_init", c_init)
val.require_finite("C_boundary", c_boundary)
```

**问题陈述**：浓度（`kg/m³` 或 `mol/m³`）必须非负。当前只校验 finite，传入 `C_init=-1.0` 会通过校验，erfc 求解器照常输出数值，`validity.passed=True`。这是物理约束未拦截，与 PR #20 Validator 设计目标矛盾。

**建议修复**：
```python
val.require_positive("C_init", c_init, allow_zero=True)
val.require_positive("C_boundary", c_boundary, allow_zero=True)
```

**对应测试**：
```python
def test_negative_concentration_flagged(self):
    out = mf_m01.solve({
        "C_init": -1.0, "C_boundary": 0.0,
        "time": 100.0, "x_position": 0.001,
        "D_eff": 1e-10,
    })
    self.assertFalse(out["validity"]["passed"])
```

**昨日 Opus-only 为何漏掉**：审查时只检查 `require_*` 是否在场，未对照 yaml 单位语义（mol/m³ → 隐含非负）。D69 双审下 GPT 5.5 直接抓到。

### P1.2 — T01/M01 thickness 无正数/有限性校验

**文件:行号**：`engine/solver/mf_t01.py:87`、`engine/solver/mf_m01.py:48` 区域

```python
# T01:
thickness   = params.get("thickness")
# 无 require_positive / require_finite 调用
# Line 118 只 isinstance 类型守卫
if isinstance(thickness, (int, float)) and isinstance(x_pos, (int, float)):
    if x_pos > thickness:  # NaN 比较恒为 False
```

**问题陈述**：
- yaml `MF-T01.runtime_variables = [T_init, T_boundary, thickness, time, x_position]` ——`thickness` 是必需 runtime variable。solver 实际上把它当 optional：`params.get("thickness")` 返回 `None` 时跳过半无限假设校验，`validity.passed=True`。
- `thickness = 0`：`x_pos > 0` 间接报错，但语义错位（应该是"thickness 非法"，不是"x 超出"）；`thickness/2 = 0` 进入 penetration heuristic 后 `penetration >= 0` 几乎必然成立，触发"semi-infinite violated" 错误，仍是错误的报错路径。
- `thickness = NaN`：所有 `>` 比较为 False → 校验全部跳过 → `validity.passed=True`。
- `thickness = inf`：penetration 永远 < inf/2 → 半无限"通过" → `validity.passed=True`，但物理无意义。
- `thickness < 0`：`x_pos > -1` 为 True → 报错"x_position exceeds slab thickness"，但本应报"thickness must be positive"。

M01 同结构（`engine/solver/mf_m01.py` 围绕 `thickness = params.get("thickness")` 的逻辑），相同问题。

**建议修复**（推荐：把 thickness 当作 yaml 声明的 required runtime variable）：
```python
thickness = params.get("thickness")
if thickness is None:
    val.issues.append(
        "thickness is a required runtime variable per MF-T01.yaml; "
        "cannot validate semi-infinite slab assumption without it"
    )
else:
    val.require_positive("thickness", thickness)
```

**对应测试**（每个 MF 至少 4 条）：
- `thickness = 0` → `validity.passed=False`
- `thickness = -1.0` → `validity.passed=False`
- `thickness = float('nan')` → `validity.passed=False`
- `thickness = float('inf')` → `validity.passed=False`

**昨日 Opus-only 为何漏掉**：检查时确认 `thickness` 进入 penetration heuristic，没有对照 yaml 把它当 required 处理。
